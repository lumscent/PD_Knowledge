# Jira Task Maker — MyLink

Скилл для быстрого создания структурированных задач в Jira.
Работает в два шага: сначала черновик, потом создание после подтверждения.

## Три типа задач

| Тип | Префикс | Проект | Когда |
|---|---|---|---|
| **Design System** | `Design: ...` | MLDS | Компоненты, токены, паттерны ДС |
| **Product** | `[Продукт] ...` | DML | Правки в конкретном продукте |
| **Board** | `Действие: ...` | DML | Аудиты, бенчмарки, ресерчи |

### Разница между типами

| | Design System | Product | Board |
|---|---|---|---|
| **Что меняется** | Компонент / токен | Экран / фича | Ничего — результат знание |
| **Кто использует результат** | Все продукты | Один продукт | Команда |
| **Артефакт** | Обновлённый компонент | Обновлённый макет | Отчёт / таблица |
| **Срок жизни** | Долгий | Реализован и закрыт | Информационный |

## Команды

### /design [компонент] [изменение]
Заводит задачу на доску **MLDS** (Design System).

**Процесс:**
1. Проанализировать что меняется (компонент / токен / паттерн)
2. Задать уточняющие вопросы если нужно
3. Сформировать черновик по шаблону из [design-system.md](references/design-system.md)
4. Показать дизайнеру для проверки
5. После подтверждения — создать задачу в MLDS

**Примеры:**
- `/design Кнопки Добавить состояние loading`
- `/design Tabs Расширить вариант Segmented`
- `/design Токены Обновить нейтральную шкалу`

---

### /product [продукт] [экран] [проблема]
Заводит продуктовую задачу на доску **DML**.

**Процесс:**
1. Проанализировать проблему на конкретном экране продукта
2. Задать уточняющие вопросы если описание неполное
3. Сформировать черновик по шаблону из [product-task.md](references/product-task.md)
4. Показать дизайнеру для проверки
5. После подтверждения — создать задачу в DML

**Примеры:**
- `/product MyTeam Профиль Добавить иерархию у блока контактов`
- `/product MyLink Hiring Пустое состояние воронки`

---

### /board [действие] [тема]
Заводит исследовательскую задачу на доску дизайнеров **DML**.

**Допустимые действия:** Аудит, Бенчмарк, Ресерч, Анализ, Конкурентный обзор

**Процесс:**
1. Определить тип действия и scope
2. Сформировать черновик по шаблону из [board-task.md](references/board-task.md)
3. Уточнить методологию и ожидаемый артефакт
4. Показать дизайнеру для проверки
5. После подтверждения — создать задачу в DML

**Примеры:**
- `/board Аудит UX/UI продукта MyTeam`
- `/board Бенчмарк Онбординг в HR-системах`
- `/board Ресерч Сценарии рекрутера в ATS`

---

## Правило

Без подтверждения дизайнера задача **не создаётся**.
