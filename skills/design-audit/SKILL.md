# Design Audit — MyLink

Скилл для проведения UX/UI аудитов и проверки копирайтинга.
Работает по скриншотам или ссылке на Figma.

## Источник истины — Storybook

**URL**: `https://storybook.mylink.kz`

При аудите сверяйся с компонентами из Storybook — это продакшн-библиотека команды.

**Как использовать:**
- Определи какие компоненты задействованы на проверяемом экране
- Открой нужный компонент: `https://storybook.mylink.kz/?path=/docs/components-{name}--docs`
- Сверь варианты, состояния и пропсы с тем что видишь на макете
- Не читай всю библиотеку — только компоненты которые реально присутствуют на экране

---

## Команды

### /audit [продукт]
Полный UX/UI аудит продукта или экрана.

**Процесс:**
1. Попросить скриншоты если не приложены
2. Определить компоненты на экране и открыть их в Storybook
3. Проанализировать по чеклисту из [checklist.md](references/checklist.md)
4. Оценить каждую проблему по шкале из [severity.md](references/severity.md)
5. Сформировать отчёт по формату из [report-format.md](references/report-format.md)
6. Предложить создать задачи в Jira через `jira-task-maker`

---

### /copy-review [экран]
Аудит UX-копирайтинга на экране.

**Процесс:**
1. Попросить скриншоты если не приложены
2. Проверить по чеклисту из [copy-checklist.md](references/copy-checklist.md)
3. Показать список правок с примерами "было → стало"
4. Предложить создать задачи в Jira через `jira-task-maker`
