# Jira Tasks — скилл для дизайн-команды

Этот скилл помогает дизайнерам быстро переводить проблемы в структурированные задачи Jira.
Работа идёт в два этапа: сначала **анализ**, потом **создание задачи**.

## Конфигурация Jira

- **cloudId**: `bf61b230-67a7-4150-82eb-4981e9ddfd51`
- **Сайт**: `mycar-group.atlassian.net`
- **Проект**: `DML` (Design MyLink)
- **Тип задачи**: `Задача` (Task)
- **Designer approve**: всегда `{ "id": "10345" }` (Yes)
- **Приоритеты**: High / Medium / Low

## Команды

### `/audit [продукт]`
Проводит UX/UI аудит продукта и создаёт задачу в Jira.

**Пример**: `/audit MyTeam`

**Что делает Claude:**
1. Запрашивает скриншоты или открывает сайт через браузер
2. Анализирует по чеклисту: визуальная консистентность, UX, копирайтинг, компоненты
3. Формирует структурированное описание задачи
4. Уточняет исполнителя и приоритет
5. Создаёт задачу в Jira

---

### `/ux-task [экран] [описание проблемы]`
Оформляет UX-проблему как задачу в Jira.

**Пример**: `/ux-task Профиль сотрудника Кнопки действий не имеют иерархии`

**Что делает Claude:**
1. Анализирует проблему, задаёт уточняющие вопросы если нужно
2. Оформляет по шаблону (см. templates/ux-task.md)
3. Предлагает приоритет и лейблы
4. Создаёт задачу в Jira после подтверждения

---

### `/tech-debt [компонент] [описание]`
Быстрое заведение технического долга по дизайну.

**Пример**: `/tech-debt Кнопки Дублирующиеся hover-стили в разных компонентах`

**Что делает Claude:**
1. Оформляет по шаблону (см. templates/tech-debt.md)
2. Автоматически ставит лейбл `tech-debt`
3. Создаёт задачу в Jira после подтверждения

---

### `/copy-review [экран]`
Аудит UX-копирайтинга на экране и создание задачи с правками.

**Пример**: `/copy-review Онбординг`

---

## Процесс работы (обязательно)

Клод **всегда** работает в два шага:

### Шаг 1 — Анализ
- Изучить проблему
- Задать уточняющие вопросы если описание неполное
- Сформировать черновик задачи по шаблону
- Показать дизайнеру для проверки

### Шаг 2 — Создание
- Получить подтверждение от дизайнера
- Уточнить исполнителя (если не указан)
- Создать задачу в Jira
- Вернуть ссылку на тикет

Без подтверждения задача **не создаётся**.

## Формат задачи в Jira

См. шаблоны в папке `templates/`.
