# Шаблон: /audit

## Название задачи
`[Аудит] UX/UI аудит продукта {название}`

## Описание

### Цель
Провести комплексный UX/UI аудит продукта {название} для выявления проблем с usability, визуальной консистентностью и соответствием дизайн-системе.

### Scope аудита
- Экраны: {список экранов или «все основные экраны»}
- Платформа: {web / mobile / оба}

### Чеклист аудита

**1. Визуальная консистентность**
- [ ] Соответствие компонентов дизайн-системе (цвета, типографика, отступы)
- [ ] Отклонения от токенов и стилей
- [ ] Несоответствия Figma vs production

**2. UX и usability**
- [ ] Ключевые user flows пройдены как пользователь
- [ ] Навигация, иерархия информации, читаемость
- [ ] Пустые состояния, ошибки, загрузка
- [ ] Мобильная адаптация

**3. UX-копирайтинг**
- [ ] Кнопки, подсказки, ошибки, заголовки
- [ ] Опечатки и непоследовательные формулировки
- [ ] Соответствие tone of voice

**4. Компоненты**
- [ ] Все состояния компонентов (hover, active, disabled, error)
- [ ] Консистентность между экранами

### Формат результатов
- Таблица найденных проблем: экран / описание / критичность / скриншот
- Группировка по типу: визуальные, UX, копирайтинг, тех долг
- Итоговый отчёт с приоритизацией

### Ожидаемый результат
- Таблица с проблемами
- Приоритизированный список правок
- Рекомендации по следующим шагам

## Лейблы
`audit`, `design-qa`

## Приоритет
High (по умолчанию для аудитов)
